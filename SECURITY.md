# Security Policy

Please use GitHub private vulnerability reporting for security-sensitive
problems. Do not open a public issue containing exploit details, credentials,
private models, personal data, or proprietary assets.

Include the affected release, impact, reproduction steps, and the smallest
shareable test input. Reports concerning untrusted glTF/GLB parsing, image
decoding, resource limits, path handling, graphics-resource lifetime, or public
API error handling are especially useful.

Ordinary product bugs and documentation problems should use GitHub Issues.
